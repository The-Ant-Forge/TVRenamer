Subtitle Merging

TVRenamer can locate sibling subtitle files (.srt, .ass, .ssa, .vtt) and merge them as soft subtitle tracks into the renamed media file. This eliminates loose subtitle files alongside your media library and gives every player native support without external .srt companions.

Enabling Subtitle Merging

  1. Open Preferences → General
  2. Check "Merge sibling subtitle files into renamed media"
  3. Pick a Default language from the dropdown (used when a subtitle filename has no language tag of its own).
  4. Optionally enable "Delete subtitle files after successful merge" to remove the original .srt/.ass/.vtt after a confirmed merge + move.
  5. Save preferences.

Subtitle merging happens automatically when files are renamed/moved. The merge runs against the source file (typically a fast local disk) before the move, then the move proceeds normally — only the final merged file traverses any network share. Metadata tagging (if enabled) likewise runs locally before the move.

While each file is being merged, the row's status cell shows a percentage counter so you can watch progress in real time. The bottom progress bar advances continuously across the whole rename action (rename + tag + move + merge), in proportion to how many sub-operations have completed.

Required External Tools

MP4 / M4V

Requires MP4Box (part of the GPAC project).

  1. Download GPAC from gpac.io
  2. Install using the Windows installer
  3. Default location: C:\Program Files\GPAC

TVRenamer automatically detects MP4Box.exe if installed in the default location or anywhere on your PATH.

MKV

Requires mkvmerge (part of MKVToolNix — the same package that powers MKV metadata tagging).

  1. Download from mkvtoolnix.download
  2. Install using the Windows installer
  3. Default location: C:\Program Files\MKVToolNix
Note: If neither tool is installed for a given format, those files are simply moved without merged subtitles. TVRenamer logs a single notice per session and skips the merge silently for the rest of the run.

How Subtitle Files Are Paired

For a media file like Westmark Academy.S01E02.mkv, TVRenamer scans the same directory for sibling subtitle files whose name starts with the media file's base name and ends with a supported subtitle extension.

Bare match

  • Westmark Academy.S01E02.srt → uses your default language preference.

Language-tagged matches

  • Westmark Academy.S01E02.en.srt → English (2-letter ISO code)
  • Westmark Academy.S01E02.eng.srt → English (3-letter ISO code)
  • Westmark Academy.S01E02.french.srt → French (full English name)
  • Westmark Academy.S01E02.pt-BR.srt → Portuguese with region tag (track named "Portuguese (BR)")

Track-flavour descriptors

Filename suffixes set the corresponding container flag where supported (mkvmerge sets per-track flags; MP4Box embeds them in the track name).

  • .sdh, .cc, .hi, .hearingimpaired → hearing-impaired
  • .forced → forced display
  • .commentary → commentary track
  • .signs, .songs, .dub → preserved as track name only

Multiple descriptors combine: Westmark Academy.S01E02.en.sdh.forced.srt → track name "English (Forced, SDH)".

Multiple subtitle files

If several subtitle files match the same media, all are merged as separate tracks. Order: language-tagged tracks first (sorted by language code), then bare matches.

Format Compatibility

Subtitle format MP4 (MP4Box) MKV (mkvmerge)
.srt (SubRip) ✓ (auto-converts to TX3G) ✓
.vtt (WebVTT) ✓ ✓
.ass, .ssa (Advanced SubStation Alpha) ✗ (skipped with warning — MP4Box rejects ASS in MP4) ✓

Idempotency — Safe to Re-run

Before merging, TVRenamer inspects the media file for existing subtitle tracks. If the container already has a track in the language we'd otherwise merge, that language is skipped — re-running the rename pipeline never adds duplicate tracks.

Failure Handling

  • Tool not on PATH: the file moves normally without merged subtitles. Logged once per session.
  • Tool error or unsupported codec: logged, the file moves with its original (external) subtitles untouched.
  • Atomic swap fails after merge (rare — usually transient AV/indexer locks): the merged temp file is preserved alongside the source as <source>.merging.<ext> for manual recovery. Look for it if a merge silently doesn't take effect.
  • Move fails after merge: the merged source file remains where it is — a valid playable file. The user's original sibling subtitle files are NOT deleted unless the move succeeded.

Verifying Merged Tracks

For MKV Files

  • Use MKVToolNix GUI → Header editor
  • Command line: mkvinfo file.mkv or mkvmerge --identify file.mkv

For MP4 Files

  • Command line: MP4Box -info file.mp4
  • Use MediaInfo for a friendly summary

Troubleshooting

"MP4Box not found" / "mkvmerge not found" status label

  • Verify the tool is installed
  • Check that the install directory is on your system PATH
  • Re-launch TVRenamer after a PATH change — tool detection is cached per session

Subtitle file is in the right folder but isn't being picked up

  • Verify the subtitle filename starts with the exact base name of the media file (the part before the final extension), followed by a dot, then the language tag (if any), then the subtitle extension.
  • Filename matching is case-insensitive but must be a prefix match.
  • Check TVRenamer logs (-Dtvrenamer.debug=true) for "Looking for subtitles" entries.

Media player doesn't show the new track

  • Some players cache track lists per-file — close and reopen the file
  • Refresh your library/metadata in your media manager (Plex, Jellyfin, Kodi)
